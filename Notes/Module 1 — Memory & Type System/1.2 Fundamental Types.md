# 1.2 Fundamental Types

## Strings

Strings are **reference types** but behave like value types because they're **immutable**. Every operation that looks like it modifies a string creates a new one.

### Immutability

```csharp
string name = "ESL";
name.ToUpper();          // does nothing to name — returns a new string
name = name.ToUpper();   // creates a new string, reassigns
```

Concatenating in a loop creates a new string object on every iteration — O(n²) allocations:

```csharp
// ❌ Creates 1000 string objects
string result = "";
for (int i = 0; i < 1000; i++)
    result += i;

// ✅ One mutable buffer, one final string
var sb = new StringBuilder();
for (int i = 0; i < 1000; i++)
    sb.Append(i);
string result = sb.ToString();
```

### StringBuilder

Mutable buffer for building strings. Use it when doing repeated concatenation, especially in loops.

```csharp
var sb = new StringBuilder();
sb.Append("Tournament: ");
sb.AppendLine("ESL Pro League");
sb.Insert(0, ">> ");
sb.Remove(0, 3);
string final = sb.ToString();
```

Rule of thumb: 3+ concatenations in a loop → use `StringBuilder`.

### Interning

String literals with the same value share the same memory location. The CLR stores them in an intern pool.

```csharp
string a = "hello";
string b = "hello";
ReferenceEquals(a, b); // true — same object in intern pool

string c = new string(new char[] { 'h','e','l','l','o' });
ReferenceEquals(a, c); // false — explicitly allocated, not interned
```

### Equality

`==` on strings compares content, not references — the operator is overloaded. This is the exception to the rule that `==` on reference types compares references.

```csharp
string a = "ESL";
string b = new string(new char[] { 'E','S','L' });
a == b;                 // true — content comparison
ReferenceEquals(a, b);  // false — different objects
```

## Arrays

Fixed-size, contiguous memory allocation, O(1) index access, O(n) search.

```csharp
int[] a = new int[5];           // default 0
int[] b = { 3, 1, 4, 1, 5 };   // inline init
int n = b.Length;               // 5

Array.Sort(b);                  // O(n log n)
Array.Reverse(b);               // O(n) in-place
int idx = Array.IndexOf(b, 4);  // O(n) linear scan, -1 if not found

// Safe copy — arrays are reference types
int[] copy = (int[])b.Clone();
```

> Arrays are **reference types** — assignment copies the reference. `b[0] = 99` mutates the original if both variables point to the same array.

```csharp
int[] a = { 1, 2, 3 };
int[] b = a;
b[0] = 99;
// a[0] is now 99 — same object in memory
```

## Enums

Named constants backed by an integer. Underlying type is `int` by default.

```csharp
public enum TournamentStatus
{
    Setup = 1,
    GroupsInProgress = 3,
    Finished = 7,
    Cancelled = 8
}
```

### [Flags] — combining values

Use powers of 2 so each value occupies a unique bit.

```csharp
[Flags]
public enum Permissions
{
    None  = 0,
    Read  = 1,  // 001
    Write = 2,  // 010
    Admin = 4   // 100
}

var p = Permissions.Read | Permissions.Write; // combine with |
p.HasFlag(Permissions.Read);                  // true — check with HasFlag
```

### Common operations

```csharp
var status = Enum.Parse<TournamentStatus>("Finished"); // throws if invalid
Enum.TryParse("Cancelled", out TournamentStatus result); // safe parse
var all = Enum.GetValues<TournamentStatus>();

int raw = (int)TournamentStatus.Finished; // 7
var s = (TournamentStatus)7;              // Finished
```

> Casting an invalid int to an enum does **not** throw — you get an undefined value. Use `Enum.IsDefined()` to validate.

## System.Object

Every type in .NET inherits from `System.Object`. It provides 4 core methods.

### ToString()

Virtual. Default returns the fully qualified type name — override for meaningful output.

```csharp
class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public override string ToString() => $"{Name}, {Age} years old";
}

var p = new Person { Name = "Alice", Age = 30 };
Console.WriteLine(p); // "Alice, 30 years old" (ToString() called implicitly)
```

### Equals(object)

Virtual. Default: reference types → reference equality; value types → value equality (uses reflection, slow).

```csharp
class Product
{
    public int Id { get; set; }

    public override bool Equals(object obj)
    {
        if (obj is not Product other) return false;
        return Id == other.Id;
    }

    // MUST override GetHashCode when overriding Equals
    public override int GetHashCode() => Id.GetHashCode();
}
```

**CRITICAL RULE:** Override `Equals()` and `GetHashCode()` together, or neither. Breaking this corrupts `Dictionary<T>`, `HashSet<T>`.

### GetHashCode()

Virtual. Returns an `int` used by `Dictionary`, `HashSet` for fast bucket lookups.

**Contract:**
1. If `a.Equals(b)` → `a.GetHashCode() == b.GetHashCode()` must be true
2. Reverse not required (collisions are allowed)
3. Hash must remain stable for the object's lifetime

```csharp
public override int GetHashCode() => HashCode.Combine(Id); // C# 9+ preferred
```

### GetType()

**Non-virtual** (cannot be overridden). Returns the exact runtime `Type`.

```csharp
object obj = "hello";
Console.WriteLine(obj.GetType().Name); // "String"

// Prefer pattern matching over GetType() for checks
if (obj is string s)
    Console.WriteLine(s.Length);
```

### Equals vs == vs ReferenceEquals

| Scenario | Equals() | == | ReferenceEquals() |
|---|---|---|---|
| Value types | value equality | value equality | false (boxing) |
| Reference types (default) | reference equality | reference equality | reference equality |
| `string` | content equality | content equality (overloaded) | depends on interning |
| Custom class (overridden) | custom logic | depends on overload | always reference |

## Interview Traps

- **"Is string a value or reference type?"** — Reference type. Behaves like value type because immutability prevents observable shared mutation.
- **"What does string immutability mean for performance?"** — `+=` in a loop is O(n²). Use `StringBuilder` for repeated modifications.
- **"Difference between `==` and `ReferenceEquals` on strings?"** — `==` compares content (overloaded). `ReferenceEquals` checks memory address.
- **"Are enums value or reference types?"** — Value type, backed by int.
- **"What happens if you cast an invalid int to an enum?"** — No exception, undefined value. Use `Enum.IsDefined()` to validate.
- **"What happens if you override only `Equals()`?"** — `Dictionary` and `HashSet` break because equal objects may hash to different buckets.
- **"Why is `GetType()` not virtual?"** — It must return the exact runtime type. If overridable, subclasses could lie about their type.
