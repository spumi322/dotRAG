# 5.2 Parameter Modifiers

By default, value types are passed **by value** — the method receives a copy. These modifiers change that behaviour.

---

## `ref` — pass by reference, must be initialized before the call

Both the caller and the method share the same variable. The method can read and modify it.

```csharp
void Double(ref int number)
{
    number *= 2; // modifies the original variable
}

int x = 5;
Double(ref x);
Console.WriteLine(x); // 10
```

**Rule:** Caller **must** initialize the variable before passing it. The method may or may not modify it.

---

## `out` — pass by reference, must be assigned inside the method

The method is responsible for assigning the variable before returning. Caller does not need to initialize it first.

```csharp
bool TryParse(string input, out int result)
{
    return int.TryParse(input, out result);
}

if (TryParse("42", out int value))
    Console.WriteLine(value); // 42

// C# 7+ — declare inline at the call site
if (int.TryParse("42", out int parsed))
    Console.WriteLine(parsed);
```

**Pattern:** `TryXxx` methods throughout the BCL (`int.TryParse`, `Dictionary.TryGetValue`, `Guid.TryParse`).

---

## `in` — pass by reference, read-only

Passes by reference for **performance** (avoids copying large structs) but prevents the method from modifying the value.

```csharp
double CalculateDistance(in Point a, in Point b)
{
    // a.X = 0; // ❌ Compile error — cannot modify an `in` parameter
    return Math.Sqrt(Math.Pow(b.X - a.X, 2) + Math.Pow(b.Y - a.Y, 2));
}
```

**When to use:** Large `readonly struct` parameters where copying is expensive but you never need to modify the value.

---

## `params` — variable number of arguments

Allows a method to accept any number of arguments of the same type. The caller passes them as a comma-separated list; the method receives them as an array.

```csharp
public int Sum(params int[] numbers)
{
    int total = 0;
    foreach (var n in numbers)
        total += n;
    return total;
}

// Call with any number of arguments
Sum(1, 2, 3);           // 6
Sum(10, 20, 30, 40);    // 100
Sum();                   // 0 — valid, empty array

// Can also pass an array directly
int[] values = { 1, 2, 3 };
Sum(values);
```

**Rules:**
- `params` must be the **last** parameter in the method signature
- Only one `params` parameter per method
- The element type can be any type (including `object` for mixed types, though avoid this)

```csharp
// Practical example — logging with string.Format style
public void Log(string format, params object[] args)
{
    Console.WriteLine(string.Format(format, args));
}

Log("User {0} created tournament {1}", userId, tournamentName);
```

---

## Modifier Comparison

| Modifier | Must init before call | Must assign in method | Modifiable | Use case |
|---|---|---|---|---|
| *(none)* | ✅ | — | ❌ (copy) | Default — pass a copy |
| `ref` | ✅ | ❌ | ✅ | Read and/or modify caller's variable |
| `out` | ❌ | ✅ | ✅ | Return additional values from a method |
| `in` | ✅ | ❌ | ❌ | Avoid copy of large struct (read-only) |
| `params` | — | — | Array body | Variable-length argument list |

---

## Interview Traps

- **"Difference between `ref` and `out`?"** — Both pass by reference. `ref` requires the variable to be initialized before the call; `out` requires the method to assign it before returning. Use `out` when the method produces a value; use `ref` when the method reads and modifies an existing value.
- **"When would you use `in`?"** — For large `readonly struct` parameters to avoid the copy cost without allowing modification. Rarely needed in everyday code.
- **"Can you have multiple `params` parameters?"** — No. Only one per method, and it must be the last parameter.
- **"Can `ref`/`out` be used with reference types?"** — Yes. With reference types, passing without `ref` already shares the object (the reference is copied). Using `ref` with a reference type lets the method reassign the caller's variable to point to a different object.
