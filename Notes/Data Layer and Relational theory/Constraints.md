Constraints collectively maintain data consistency and support database integrity.

1. **Primary Key Constraint:** Ensures unique identification of records.
   
2. **Foreign Key Constraint:** Links tables by connecting a column to another table's primary key.
   
3. **Unique Constraint:** Requires unique values in a column or set of columns.
   
4. **Check Constraint:** Restricts allowed values in a column.
   
5. **Default Constraint:** Sets a default value for a column.
   
6. **Not Null Constraint:** Forbids null values in a column.
   
7. **Cascade Constraint:** Defines actions when referenced records are updated or deleted.
   
8. **Index Constraint:** Speeds up data retrieval by creating an index on specified columns.
