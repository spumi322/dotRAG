Software architecture is the blueprint for how a software system is structured. It's necessary because it organizes code and components in a way that makes the system easier to understand, maintain, and expand.

**Anemic vs Rich model:**
Anemic model only contains the props of a plain object, without the business logic(DTOs), while Rich model has both the data and the behavior. Anemic clearly separates the operations from data, while rich encapsulates them.
