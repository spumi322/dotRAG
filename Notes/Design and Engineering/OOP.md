### *OOP (Object-Oriented Programming):*

- **Definition:** A programming paradigm centered around the concept of "objects," which encapsulate data and behavior.

**OOP (Object-Oriented Programming):**
    - Focuses on objects with state and behavior.
    - Emphasizes data encapsulation.
    - Utilizes classes and objects
    - Supports concepts like inheritance and polymorphism.

**Why Use OOP:**
- **Modularity:** Encourages modular design, making code more organized and maintainable.
- **Reusability:** Facilitates reuse of code through inheritance and composition.
- **Flexibility:** Enhances code flexibility and adaptability to changes.
- **Readability:** Improves code readability and understanding.

**Four Pillars of OOP:**

1. **[[Encapsulation]]:** Bundling data and methods that operate on the data into a single unit (class/object).
   Think of a TV remote. You have buttons (methods) to change the channel or volume (data), but you don't need to know how the circuits inside work (the data and methods are bundled together and hidden from you).
2. **[[Inheritance]]:** Allows a class to inherit properties and behaviors from another class, promoting code reuse.
   Imagine a basic phone and a smartphone. The smartphone inherits features from the basic phone, like making calls and sending texts, but it also has its own additional features like internet browsing and apps.
3. **[[Polymorphism]]:** Letting different types of objects be treated as if they were the same type.
   Consider a drawing app where you can use different tools like a pen, brush, or eraser. Even though they're different tools, you use them in the same way by moving with the cursor across the screen.
4. **[[Abstraction]]:** Focusing on what an object does instead of how it does it, highlighting the essential qualities while hiding the specific implementation details.
   Think of a car. You know it can drive, stop, and turn without needing to understand how the engine works, how the brakes apply force, or how the steering changes the wheels' direction. You're using a simplified view of the car, focusing only on what it does, not how it does it.

### *Difference Between OOP and Functional:*

"OOP = objects with state and behavior. Functional = pure functions, no side effects, immutable data. C# supports both — LINQ is functional style, classes are OOP."
